**Evidence:** 6007 opportunities/60s produced vs 2784 executions/60s processed
**Recommendation:**
- SHORT TERM: Increase `MAXLEN` for execution-requests from 25K to 100K (buys buffer time)
- SHORT TERM: Lower backpressure threshold from 0.8 to 0.5 (throttle coordinator sooner)
- MEDIUM TERM: Profile EE processing to find bottleneck (200ms blockMs? validation overhead?)
- LONG TERM: Horizontal scaling — multiple EE instances with partition-based assignment

### 2. execution-results Stream at Cap (RT-ER-CAP)
**Impact:** Coordinator may miss execution outcome data, leading to stale success rate metrics.
**Recommendation:** Increase MAXLEN from 5K to 25K to match execution-requests capacity.

**Recommendation:** Implement the 6 v3.0 BI metrics in execution engine.

### 4. Cross-Chain Detector Metrics Gap (RT-CC-METRICS)
